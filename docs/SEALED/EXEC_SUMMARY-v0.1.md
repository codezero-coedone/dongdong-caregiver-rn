# EXEC SUMMARY — v0.1 (SEALED)

대상: 대표/PM  
목적: 현재 상태/결정사항/리스크/다음 액션을 “1페이지”로 고정한다.

---

## 1) 지금 무엇이 “확정”인가 (결정사항)

- **빌드/배포 단일 레일**: 로컬 빌드 금지 → `코드 수정 → push → Codemagic → AAB/APK 산출 → (자동) Play 업로드`
- **인증 범위(v0.1)**: **Kakao 고정** (Apple/Google은 Out-of-scope)
- **플로우 경계선(공통, 분기 금지)**: `온보딩 → 권한 → 카카오 → SMS → 인증완료`
- **소유권**: **토큰/인증 소유권은 RN 앱이 단일 소유**, WebView는 **주입(inject)만**
- **로컬 코어 서버**: **127.0.0.1:8001 절대 중단 금지**

---

## 2) 우리가 해결한 “병목” (왜 속도가 올라가나)

- **로컬 환경 지옥(권한/락/툴체인) 제거**: CI 중심으로 결정성 확보
- **관제(Observability) 확보**: 앱 내부 DBG로 400/401/500/NETWORK를 즉시 분리(추측 제거)
- **SSOT(SEALED)로 경계선 봉인**: “같은 문제 24시간 반복” 구조적 재발 방지

---

## 3) 현재 남은 리스크 (v0.1 범위에서 관리)

- **Guardian 업로드 키/Play App Signing 정합성**: 업로드 키 reset 절차가 필요할 수 있음(워크플로우/아티팩트로 대응)
- **카카오 KeyHash 등록**: Upload Key + Play App Signing SHA1 기반 KeyHash가 둘 다 필요
- **WebView/Backend 연동 이슈**: DBG로 endpoint 단위로 즉시 분리 후 수정
